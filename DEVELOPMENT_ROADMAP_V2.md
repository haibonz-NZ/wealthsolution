# 全球财富风险自查系统 (GWRC) - 深度开发规划书

**基于文档**：《跨境财富智能AI 顾问助手系统开发需求（基础版）.pdf》
**当前状态**：V1.0 视觉原型已上线 (Visual Prototype Live)
**下一阶段目标**：从“好看的 H5”进化为“可用的获客引擎”。

---

## 📅 开发分期规划 (Development Phases)

为了确保系统稳健落地，建议将剩余功能分为三个阶段进行开发：

### 🚀 Phase 1: 视觉与交互基础 (已完成 ✅)
*   **核心模块**：Landing 页、问答录入流、五维雷达图、一键修复动效、留资弹窗。
*   **现状**：界面已高度还原 UI 设计，流程通畅，但数据是静态/模拟的，尚未与后台打通。

### 🔌 Phase 2: 数据闭环与线索流转 (Data Loop) —— **🔥 优先级：最高**
*   **目标**：让客户在前台留下的电话和资产数据，真正流转到顾问的后台。
*   **功能细化**：
    1.  **线索推送 API**：开发一个无需鉴权的公开接口（受限频次），接收前台提交的 `WizardData`（身份、资产、痛点、手机号）。
    2.  **后台接收**：在 `wealth-solution-web` (顾问端) 的 Dashboard 增加 **“新线索 (Leads)”** 卡片。
    3.  **顾问通知**：(可选) 邮件或简单的红点通知，告知顾问“有新客户完成了体检”。
    4.  **关联逻辑**：根据 URL 中的 `advisor_id` 参数，自动将线索分配给对应顾问。

### 🧠 Phase 3: AI 智能增强 (AI Intelligence) —— **🌟 核心差异化**
*   **目标**：让报告的诊断结果是“真 AI 算出来的”，而不是写死的逻辑。
*   **功能细化**：
    1.  **动态风险引擎**：
        *   接入 DeepSeek API。
        *   将客户勾选的“身份+资产”发给 AI，让 AI 判断是红灯还是绿灯，并生成一句话简评。
    2.  **“小灯泡” RAG 解释**：
        *   点击风险项旁边的“💡”，AI 实时调取（或生成）对应的法律/税务解释（如“根据美国税法...”）。
    3.  **相似案例匹配 (Real Case Matcher)**：
        *   建立一个包含 5-10 个典型脱敏案例的 JSON 库。
        *   根据客户标签（如“美籍”、“房产”），动态筛选最匹配的案例展示，而不是永远显示“刘先生”。

### 🎨 Phase 4: 体验极致化 (UX Polish) —— **💎 锦上添花**
*   **目标**：达到 PDF 要求的“感官震慑”极致细节。
*   **功能细化**：
    1.  **iPad 双栏适配**：实现 PDF 要求的“左侧地图/右侧选项”分栏布局。
    2.  **损益模拟器升级**：
        *   增加“离境税”计算器（针对加/澳身份）。
        *   增加具体的金额跳动动画（不仅是进度条）。
    3.  **PDF 报告导出**：
        *   客户点击下载时，生成一份带水印、带顾问名片的精美 PDF 简报（作为留资诱饵）。

## 🛠️ 详细开发需求清单 (Backlog)

### 1. 损益模拟器增强 (Simulator Upgrade)
*   **需求**：目前仅展示了代际衰减。需增加“离境税”场景。
*   **逻辑**：
    *   IF (身份包含 CA/AU) -> 显示“离境清算税 (Departure Tax)”预估条。
    *   数值算法：(资产总额 * 增值部分假设 50%) * 25% 税率。

### 2. 相似案例库 (Case Library)
*   **需求**：动态匹配案例。
*   **数据结构**：
    ```typescript
    interface SuccessCase {
      tags: ['US', 'RealEstate', 'Trust'];
      title: "张女士（北京/旧金山）";
      painPoint: "美籍子女继承国内房产";
      saving: "240万美金";
    }
    ```
*   **逻辑**：前端根据用户选的 tag，从库里 `filter` 出重合度最高的案例。

### 3. “小灯泡” 知识库 (Knowledge Tip)
*   **交互**：在“风险红绿灯”列表中，每条风险增加一个可点击的 icon。
*   **效果**：点击后展开一个半透明的气泡，显示 AI 生成的专业解释（约 100 字）。

### 4. 顾问名片动态化 (Dynamic Advisor Card)
*   **需求**：Landing 页底部的顾问信息不能写死。
*   **逻辑**：
    *   前台读取 `?advisor_id=123`。
    *   请求后台 API `/api/advisor/123` 获取该顾问的真实头像、姓名、头衔。
    *   如果 ID 无效，显示“The One 官方顾问团队”。

## 🚦 建议执行顺序

1.  **立即执行 (Phase 2)**：**打通数据**。没有数据回流，前台只是个玩具。我们需要让顾问在后台看到客户填了什么。
2.  **随后执行 (Phase 3)**：**接入 AI**。让诊断结果更真实、更唬人。
3.  **最后执行 (Phase 4)**：**优化细节**。

**您是否同意先进行 Phase 2（前后端数据打通）的开发？**
这将涉及修改后台 (`wealth-solution-web`) 以接收数据，并修改前台 (`wealth-solution-client`) 以发送数据。
